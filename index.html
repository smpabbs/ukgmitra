2 Studi Kasus

📖 Cerita

Pak Fahim adalah seorang guru IPA kelas 8 di SMP IT Al Firdaus. Beliau mengalami kesulitan saat menjelaskan materi proses penutupan luka (wound healing) kepada siswa. Materi ini melibatkan banyak istilah dan mekanisme biologis yang abstrak — seperti peran trombosit, sel darah putih, fibrin, serta ancaman bakteri dan virus — sehingga siswa sulit membayangkan bagaimana proses ini terjadi di dalam tubuh.

Pak Fahim ingin media yang interaktif dan visual agar siswa bisa melihat langsung bagaimana luka sembuh dan apa yang terjadi jika ada infeksi. Beliau kemudian meminta bantuan kepada Anda, guru ICT yang memahami Construct 3.

Tugas Anda: Buatlah game/simulasi interaktif menggunakan Construct 3 yang dapat memudahkan siswa kelas 8 memahami proses penutupan luka, lengkap dengan elemen penyembuh dan ancamannya.

3 Alur Gameplay

Game berjalan real-time. Berikut alur dari mulai hingga menang/kalah:

MULAI │ ▼ ┌─────────────────────────────┐ │ Trombosit auto-spawn │ │ (setiap ~3 detik dari tepi) │ └─────────────┬───────────────┘ │ Bergerak otomatis ke luka ▼ ┌─────────────────────────────┐ │ Trombosit sampai di luka │ │ ──► Progress penutupan +5% │ └─────────────────────────────┘ ┌─────────────────────────────┐ │ Sel Darah Putih │ │ (dikontrol PEMAIN) │ │ mouse / keyboard / touch │ └─────────────┬───────────────┘ │ ┌───────┴───────┐ ▼ ▼ Bakteri Virus │ │ ├─ kena luka ├─ kena player │ → Infeksi + │ → Nyawa -1 │ │ └─ tertangkap └─ tertangkap SDP → skor + SDP → skor + ⚡ Power-up (fibrin) muncul di milestone 25%, 50%, 75% Ambil → progress tambahan cepat ┌───────────┴───────────┐ ▼ ▼ ┌──────────────┐ ┌──────────────┐ │ Progress │ │ Infeksi │ │ = 100% │ │ = 100% │ │ │ │ ATAU │ │ 🏆 MENANG │ │ Nyawa = 0 │ └──────────────┘ │ │ │ ❌ KALAH │ └──────────────┘

4 Ketentuan Ujian

✅ BOLEH
  • Menggunakan akun Construct 3 free tier
  • Menggunakan sprite dari Asset Store gratis atau buat sendiri
  • Menambahkan fitur di luar mekanik minimal (power-up, level, animasi, suara, partikel)
  • Berkreasi bebas dalam desain, warna, dan tata letak
  • Menggunakan keyboard arrow keys sebagai alternatif kontrol mouse
  • Mencari referensi online selama pengerjaan
  • Menggunakan AI (ChatGPT, Claude, Copilot, dll) — wajib melampirkan log percakapan dalam bentuk PDF
❌ TIDAK BOLEH
  • Menggunakan template project yang sudah jadi (dari nol)
  • Copy-paste event sheet dari peserta lain
  • Menggunakan asset berbayar tanpa lisensi
  • Mengirim project yang tidak bisa di-preview (error)
  • Menitipkan file ke peserta lain
  • Mengumpulkan setelah batas waktu yang ditentukan
